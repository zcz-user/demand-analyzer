# 📖 使用示例

以下展示 Demand Analyzer 7 部分拆解框架在不同场景下的实际应用。

---

## 场景 1：用户提出任务

> **用户：** "帮我查一下某公司最近发生了什么"

### 拆解输出

```
1️⃣ 核心意图
用户想了解某公司的近期动态，可能为投资、合作或研究做准备。

2️⃣ 关键约束
- 时效性要求高（"最近"暗示近期事件）
- 可能需要对比多源信息
- 信任度链条很重要

3️⃣ 隐含需求
- 用户可能想了解事件背后的深层逻辑
- 可能需要判断是利好还是利空
- 可能需要与其他公司对比

4️⃣ 歧义澄清
您说的"某公司"是指：
A. 您关注的公司（请提供名称）
B. 一个我该知道的/热门公司
C. 还是让我推荐？

5️⃣ 信息缺口
- 具体的公司名称
- "最近"的具体时间范围（3 天/1 周/1 月）
- 关注的领域（财务/产品/人事/战略）

6️⃣ 可选方案
A. ⭐⭐⭐⭐⭐ 全景分析（股价变动 + 重大事件 + 行业影响）
B. ⭐⭐⭐⭐ 快速摘要（只给 3 条最重要信息）
C. ⭐⭐⭐ 指定维度深度分析

7️⃣ 执行计划
确认公司名称和关注维度后，搜索并整理信息输出。
```

---

## 场景 2：模糊表达

> **用户：** "那个框架搞一下"

### 拆解输出

```
1️⃣ 核心意图
用户想让我执行或应用某个框架，但未明确指代。

2️⃣ 关键约束
- 用户默认我应该知道"那个"指什么
- 猜错会浪费时间

3️⃣ 隐含需求
- 用户希望我能记住上下文
- 用户不想重复说明

4️⃣ 歧义澄清
您说的"框架"是指我们讨论过的：
1. 7 部分需求拆解框架
2. 横纵分析法
3. KR 工作流
4. 还是其他？

5️⃣ 信息缺口
- "搞一下"的具体含义（执行/复习/修改/分享）
- 如果和无关，需要重新获取上下文

6️⃣ 可选方案
A. ⭐⭐⭐⭐⭐ 执行指定框架
B. ⭐⭐⭐⭐ 先回顾框架内容再讨论
C. ⭐⭐⭐ 我用语义搜索查查最近讨论了什么

7️⃣ 执行计划
请确认是哪个框架和具体操作~
```

## 场景 3：用户分享链接

> **用户：** [分享链接] "这个你帮我看看"

### 拆解输出

```
1️⃣ 核心意图
用户想让我评估或分析分享的内容。

2️⃣ 关键约束
- 内容可能来自特定平台（News/虎嗅/知乎/公众号）
- 有时效性要求

3️⃣ 隐含需求
- 用户想确认这个信息的可靠性
- 用户想获取更深层的洞察
- 用户可能想让我基于此做延伸研究

4️⃣ 歧义澄清
您希望我：
A. 📖 内容摘要 + 可靠性评估
B. 🔍 延伸研究 + 补充信息
C. ⚖️ 正反观点分析
D. 💡 基于此给出建议

5️⃣ 信息缺口
需要先阅读链接内容才能细化方案。

6️⃣ 可选方案
A. ⭐⭐⭐⭐⭐ 先阅读，再按您的要求输出
B. ⭐⭐⭐⭐ 快速扫读 + 关键信息提取
C. ⭐⭐⭐ 外部搜索补充信息后综合输出

7️⃣ 执行计划
先阅读链接内容，然后按 A 方案执行。
```

---

## 模式总结

无论用户说什么，流程始终如一：

```bash
用户消息 → ⏸️ 停顿3秒 → 7部分拆解 → ✅ 用户确认 → 🚀 执行
```

关键在于：**不猜、不急、系统性追问。**
